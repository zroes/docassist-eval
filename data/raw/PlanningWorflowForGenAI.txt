          <p>A project is a collaborative workspace where people work with models and data to fulfill a shared goal.</p>
          <p>You need a project to build prompts, run experiments, and tune models.</p>
          <p>A project for a gen AI solution typically contains the following types of items that are either explicitly added by AI engineers or are created as the result of a process:</p>
          <ul class="cds--list--unordered">
            <li class="cds--list__item">Connection assets to data sources, such as a vector store or where you store training or tuning data.</li>
            <li class="cds--list__item">Data assets that represent datasets for training or tuning models.</li>
            <li class="cds--list__item">Prompt session assets that you save for future reference.</li>
            <li class="cds--list__item">Prompt template assets that provide endpoints for inferencing.</li>
            <li class="cds--list__item">Notebooks that you create or that are generated by processes such as saving a prompt as a notebook or running an AutoAI experiment.</li>
            <li class="cds--list__item">Vector indexes that represent vectorized documents for a RAG pattern.</li>
            <li class="cds--list__item">Experiment and flow assets that you create from running tools, such as AutoAI, Tuning Studio, or Synthetic Data Generator.</li>
            <li class="cds--list__item">AI service assets that provide endpoints for gen AI patterns, such as RAG.</li>
            <li class="cds--list__item">Jobs that are created by running assets in tools.</li>
          </ul>
          <p>You have an automatically created sandbox project. However, you might want to create a project with a name that reflects your goal. You can create a project from the home page or from the navigation menu. Add everyone who you want to work on
            the solution. You assign a role to each collaborator to control their permissions in the project.</p>

          <p>Data preparation involves providing access to the required data in the format and at the quality level that your solution needs.</p>
          <p>You need to prepare data if you plan to ground the model with your documents in a RAG pattern, to evaluate a prompt template, or to tune a foundation model. You might not need to prepare data if your use case is to translate, summarize, classify,
            or generate text, unless you want to run evalutions.</p>
          <p>For a RAG pattern, you transform documents into embedded vectors for efficient retrieval. You store vectorized documents in a vector store and retrieve them with a vector index. You can include documents in a RAG pattern in the following ways:</p>
          <ul class="cds--list--unordered">
            <li class="cds--list__item">Upload document files from your local system and add them to a vector store</li>
            <li class="cds--list__item">Specify documents that are in an existing vector store</li>
            <li class="cds--list__item">Add documents from a connected data source into a vector store</li>
          </ul>
          <p>You can choose from different methods of creating a RAG pattern, depending on the total size of your documents, the level of automation for experimenting, and other factors.</p>
          <p>For evaluating a prompt template or tuning a foundation model, you provide a data set that contains representative input to the model and the appropriate output for the model to generate in response. You can provide tuning data in the following
            ways:</p>
          <ul class="cds--list--unordered">
            <li class="cds--list__item">Upload a file from your local system</li>
            <li class="cds--list__item">Connect to the data source that contains the data set</li>
            <li class="cds--list__item">Generate synthetic data</li>
          </ul>

          <p>A prompt is how you instruct a foundation model to generate a response. You save a prompt as a prompt template. When you add documents, images, or agents to a prompt, you save it as an AI service.</p>
          <p>You can experiment with prompts by altering conditions in the following ways:</p>
          <ul class="cds--list--unordered">
            <li class="cds--list__item">Switching between chat and nonchat modes</li>
            <li class="cds--list__item">Altering your prompt text or the system prompt</li>
            <li class="cds--list__item">Changing the foundation model</li>
            <li class="cds--list__item">Adjusting the model parameters</li>
            <li class="cds--list__item">Enabling and disabling guardrails</li>
            <li class="cds--list__item">Adding images or documents to a chat</li>
            <li class="cds--list__item">Adding variables to dynamically change prompt text</li>
            <li class="cds--list__item">Configuring agents to call tools</li>
          </ul>
          <p>To develop your prompts, you can experiment in the Prompt Lab or with REST APIs, Python, or Node.js code. To automate finding an optimal RAG pattern, run an AutoAI for RAG experiment.</p>
          <p>To develop an AI agent that calls tools, you can experiment in the Agent Lab or with REST APIs, Python, or Node.js code. You can add governed agents and tools that are provided by IBM, your organization, or other providers into your solution

          <p>An evaluation of a gen AI asset, such as a prompt or AI service, tests the quality of the model output for the set of metrics that you choose. Some metrics are based on comparing model output against the appropriate output that you provide in
            the testing data set. How efficiently your model generates responses is also evaluated.</p>
          <p>Your organization might require evaluations for regulatory compliance or internal policies. However, evaluations are useful even when they are not required because the metric scores can indicate the quality of your solution and might predict
            decreased user satisfaction when scores drop.</p>
          <p>When you evaluate an AI asset, you can configure the following factors:</p>
          <ul class="cds--list--unordered">
            <li class="cds--list__item">The sample size to test</li>
            <li class="cds--list__item">Which metrics to include</li>
            <li class="cds--list__item">The threshold value for each metric</li>
          </ul>
          <p>General metrics provide the following types of information about the AI asset:</p>
          <ul class="cds--list--unordered">
            <li class="cds--list__item">How well the model performs compared to a labeled test set.</li>
            <li class="cds--list__item">Whether the model produces biased outcomes.</li>
            <li class="cds--list__item">Changes in accuracy of model output over time.</li>
            <li class="cds--list__item">How efficiently the model processes transactions.</li>
          </ul>
          <p>Metrics that are specific to generative AI provide the following types of information about the AI asset:</p>
          <ul class="cds--list--unordered">
            <li class="cds--list__item">How similar the output text is to the input text.</li>
            <li class="cds--list__item">How similar the output text is to reference outputs.</li>
            <li class="cds--list__item">Whether the input or output text contains harmful or sensitive information.</li>
            <li class="cds--list__item">How well the AI asset maintains performance against adversarial attacks.</li>
          </ul>
          <p>You view the current results and the results over time. The results of each evaluation are added to the use case for the prompt.</p>
          <p>To evaluate a prompt, run an evaluation in the Prompt Lab, with code, or from a deployed prompt template. If you run an AutoAI experiment for a RAG pattern, the candidate AI services are evaluated and ranked automatically.</p>
          <p>To evaluate and compare multiple prompt templates simultaneously, run an evaluation experiment in Evaluation Studio.</p>
